# SSMS 공통표준 용어사전 (v1.21) — DB 명명 표준 & DDL 자동 교정

> [!summary] 한 줄 요약
> `도급사업안전관리(SSMS)` 프로젝트의 DB 물리 명명 표준. 행안부 표준단어/표준용어/공통표준도메인 기반이며, 한글 의미 → snake_case 물리명을 결정하는 **유일 기준**이다. 이 표준을 지식베이스로 적재해 **DDL의 테이블·컬럼명/타입을 자동 교정**하는 기능을 RAG 앱(`localhost:8800`)에 통합했다.

## 1. 출처 파일

## 2. 사전 3종의 역할

| 사전 | 건수 | 역할 | 핵심 컬럼 |
|------|-----:|------|-----------|
| **단어사전** | 19,623 | 한글 단어 → 표준 약어(물리명). 명명의 최소 단위. | 단어 / 물리명 / 정의 |
| **용어사전** | 499 | 한글 용어(단어 조합) → 등록 물리명 + 도메인. | 용어 / 물리명 / 도메인 / 정의 |
| **도메인사전** | 65 | 데이터타입·길이 표준(PostgreSQL). | 도메인명 / 데이터타입 / 길이 |
| 용어변환툴 | — | 단어→약어 조합으로 물리명을 생성하는 예시 시트. | 단어1~5 / 약어1~5 / 완성 물리 |

> [!note] 약어는 임의로 만들지 않는다
> 약어의 **유일 기준은 단어사전의 `물리명`** 이다. 흔한 오류: `mng`✗→`mgt`✓(관리), `nm`✗→`name`✓(명), `rsk`✗→`risk`✓(위험). 개정이력 대부분이 이런 약어 표준화 이력이다.

## 3. 명명 규칙

1. 물리명은 **snake_case**, 표준단어의 약어를 `_`로 연결. (예: `위험 평가 점수` → `risk_eval_score`)
2. 한글 용어가 **용어사전에 등록**돼 있으면 그 물리명을 그대로 사용(재조합 금지). (예: `가입 상태` → `join_status`)
3. 데이터타입/길이는 **도메인사전**을 따른다.

### 도메인 분류 (그룹 → 대표 타입)
- **명칭**(20): `명_V200`=VARCHAR(200), `내용`/`설명`=TEXT
- **식별자**(20): `코드_V20`=VARCHAR(20), `*아이디_V50`=VARCHAR(50), `고유번호`=BIGINT
- **수치**(10): `수`/`순서`/`점수`=INTEGER, `금액`/`파일사이즈`=BIGINT
- **일시**(5): `일시`=TIMESTAMP, `일자`=DATE, `시간`=TIME, `연월_V7`=VARCHAR(7)
- **여부/상태**(1): `여부_V1`=VARCHAR(1)
- **연락/경로**(5): `이메일_V200`, `전화번호_V20`, `URL_V1000`, `파일경로_V1000`, `IP주소_V50`
- **보안**(3): `비밀번호_V200`, `비밀번호솔트_V200`, `초대토큰_V100`
- **특수**(1): `JSONB`

## 5. 지식베이스 적재 & DDL 자동 교정

엑셀 → SQLite/CSV로 추출해 두 곳에 적재했다.

- **RAG 앱**: `~/obsidian-rag-poc/data/ssms_terms.db` + `ssms.py` — `localhost:8800`이 DDL 질의를 감지하면 노트 RAG 대신 이 사전을 근거로 교정. ([[#6. RAG 앱 통합 동작]])
- **Claude Code 스킬**: `~/.claude/skills/ddl-corrector/` — CLI(`lookup.py`)로 사전 검색 후 교정.

### 추출 스크립트 (사전 갱신 시 재실행)
```python
# openpyxl로 단어/용어/도메인 시트를 읽어 sqlite3 + csv로 저장.
# 단어사전 header row=5, 용어사전 row=4(NO/버전/용어/물리명/도메인/정의), 도메인사전 row=5.
# 산출: ssms_terms.db(word/term/domain 테이블 + ko/phys 인덱스), *_dict.csv
# 전체 스크립트는 ~/obsidian-rag-poc/README.md 및 스킬 폴더 참고.
```

### lookup.py 검색 명령
| 명령 | 용도 |
|------|------|
| `word <한글>` / `rword <약어>` | 단어 ↔ 표준 약어 |
| `term <한글용어>` / `rterm <물리명>` | 용어 ↔ 등록 물리명 |
| `domain <도메인명>` | 데이터타입/길이 |
| `build "위험 평가 점수"` | 한글 용어 → 단어조합 물리명 생성 |
| `check risk_eval_score` | 물리 토큰 분해 → 표준 약어 여부 검증 |

## 6. RAG 앱 통합 동작

`main.py`의 LLM 호출을 **질의 종류에 따라 분기**(`plan_call`)하도록 변형:

- `ssms.is_ddl_query(q)` → `CREATE/ALTER TABLE` 또는 "물리명/표준 단어/DDL 교정" 등 의도 감지.
- DDL이면 → `ssms.build_context(q)`가 ① 식별자 토큰 검증(`check`) ② 한글 단어→약어 ③ 도메인 전량을 근거로 묶고, **교정 전용 시스템 프롬프트**로 LLM 1회 호출.
- 아니면 → 기존 Obsidian 노트 RAG 그대로.
- 응답 meta에 `mode: ddl|rag` 추가, 웹 UI에 `🛠 DDL 교정 모드` 배지 표시.

> [!tip] 한글 → DDL 흐름이 가장 정확
> 영어 약어를 거꾸로 추정하는 것보다, 한글 의미(컬럼 주석/설명)를 주면 단어사전 정확 매칭으로 표준 물리명이 결정된다. 작은 로컬 모델이 조합을 약하게 하면 `KB_MODEL=mlx-community/gemma-4-26b-a4b-it-4bit`로 교체.
