# 대화형·멀티턴 RAG — 후속 질문 contextualization

> [!summary] 한 줄 요약
> RAG 노트 대부분이 *단발 질의*를 전제하지만, 실제 챗봇 트래픽은 **후속 질문**("그건 왜?"·"더 자세히"·"그럼 환불은?")이 다수다. 후속 질의는 단독으로는 의미가 없어 — 그대로 검색하면 엉뚱한 청크가 나온다. 핵심은 **대화 이력 + 현재 질문 → standalone 질의로 재작성(contextualization)** 후 검색하는 것. 더해 ① 언제 재검색 vs 직전 컨텍스트 재사용 ② 이력 관리(요약/윈도우) ③ 화제 전환 감지가 멀티턴 품질을 가른다.

> 단독 질의 변환은 [[Query-Transformation]](HyDE·multi-query), E2E 구현은 [[RAG-Chatbot-E2E]], 단계 실패 진단은 [[RAG-Failure-Debugging]]. 이 노트는 **대화 이력을 검색 질의로 해소**하는 멀티턴 고유 축.

---

## 1. 왜 후속 질의는 그대로 검색하면 실패하는가

```
사용자: "환불 정책이 뭐야?"          → 검색 OK("환불 정책")
사용자: "그럼 기간은?"                → "그럼 기간은?"을 그대로 임베딩하면?
  → "환불 가능 기간"이라는 의도가 사라짐 → 무관한 청크 검색 → 오답
대명사·생략·지시("그거", "위에서", "더")는 이전 맥락에 의존
  → standalone(자립) 질의가 아니면 검색이 구조적으로 빗나감
```

## 2. 핵심 — 질의 재작성(Query Contextualization) ⭐

```
LLM에 [대화 이력 + 현재 질문]을 주고 "독립적으로 검색 가능한 질의"로 재작성:
  이력: Q1 "환불 정책이 뭐야?" / A1 "..."
  현재: "그럼 기간은?"
  → 재작성: "환불 가능 기간은 얼마인가?"  ← 이걸로 검색
LangChain의 "history-aware retriever", LlamaIndex의 condense-question이 이 패턴.
```

> [!tip] 재작성은 검색용, 생성은 원문 + 이력으로
> 재작성된 질의는 **검색에만** 쓴다. 답 생성에는 원래 질문·전체 이력·검색결과를 함께 넣어 자연스러운 대화 흐름을 유지. (검색 질의 ≠ 생성 프롬프트)

## 3. 재검색할 것인가 — 매 턴 검색은 낭비

```
모든 턴에 검색이 필요하진 않다:
  - 후속이 "직전 답"에 대한 것이면 → 이미 컨텍스트에 있음(재검색 불필요)
  - 새 정보가 필요한 질문이면 → 재작성 후 재검색
판단: LLM/분류기로 "검색 필요?"를 먼저 결정(라우팅) → 불필요 검색의 지연·비용 절감
함정: 항상 재검색하면 느리고, 절대 재검색 안 하면 새 주제에서 실패 → 균형
```

## 4. 대화 이력 관리 — 토큰 예산과의 싸움

```
이력이 길어지면 컨텍스트 예산 초과·비용↑·lost-in-middle:
  - 전체 유지: 짧은 대화엔 OK, 길면 폭발
  - 슬라이딩 윈도우: 최근 N턴만 — 단순하나 오래된 맥락 유실
  - 요약(running summary): 누적 대화를 요약해 압축 — 정보 손실 위험
  - 하이브리드: 요약(과거) + 최근 N턴 원문 — 실무 기본
재작성 품질은 "이력 중 무엇을 보여주느냐"에 좌우 → 요약이 부정확하면 재작성도 오염
```

## 5. ★ 함정 — 화제 전환과 이력 오염 ★

```
주제가 바뀌었는데 이력을 끌고 가면 재작성이 엉뚱해진다:
  Q1~Q3: 환불 얘기 / Q4: "배송은 며칠 걸려?"(새 주제)
  → 이력에 휘둘려 "환불 배송"으로 재작성하면 오답
대응: 화제 전환 감지(현재 질문이 이력과 무관하면 이력 비중↓ 또는 standalone 그대로)
또한:
  - 좌표 참조("위 표 3번째") → 이전 답 텍스트를 이력에 보존해야 해소 가능
  - 멀티턴 권한: 턴마다 사용자 권한 재확인(긴 세션 중 권한 변동) → [[KB-Access-Control]]
  - 재작성 LLM 호출이 매 턴 지연 추가 → 짧은 follow-up은 캐시·경량 모델로([[Prompt-Caching]])
```

## 6. 턴 흐름 정리

```
사용자 질문 → [재검색 필요?] ─아니오→ 직전 컨텍스트로 생성
                   │예
                   ▼
           [이력+질문 → standalone 재작성] → 검색/리랭킹 → [이력+검색결과+원질문] → 생성
원칙: 단발 RAG 파이프라인 앞에 "대화 해소" 단을 끼우는 것 — 나머지 검색 스택은 재사용
```
